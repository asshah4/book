7.1 Speed of for loops

Here is an issue with a function using for loops. Generally, if there is an iterative process internal to the function, it will have an \(~O(N^2)\) performance. The card::recur_survival_table is built with intuitive for loops, at a significant cost with increased N sizes.

Here is the performance of the function:
